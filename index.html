Blocktracker

Hardcoded Block Tracker for Vanilla Minecraft 1.8 servers. (API-less)

View project onGitHub

What Is BlockTracker?.

BlockTracker is a modification for the Minecraft Server. The existance of this project is drawn from struggle Minecraft servers have without unoffical API's such as Bukkit or MCP. Without those API's many programmers are incapable to create their modifications, be it the extra amount of time that is required, the extra amount of effort, or simply the level of difficulty. That's why I've created BlockTracker. A modification that latches directly into the Minecraft server's core code. As a result, this modification can exist BEFORE API's like Bukkit or MCP have been updated.

What Does BlockTracker Do?

BlockTracker ties into the core Minecraft server code. Whenever a block is broken or placed in the world, BlockTracker's hooks parse the playername, coordinates, and the blocktype from the Minecraft server and into the BlockTracker modification. BlockTracker then connects to a mySQL server configured by you (the server administrator) and inserts that information to database and table.

Future Development

BlockTracker was built in two phases over a period of one day. The inital phase was to decompile the Minecraft code and map key methods. After locating these methods it was a matter of parsing all of the variable data and processing it to a nice and clean human readable state, where it could then be inserted into a database for storage.

In the future BlockTracker will be expanded to poses a command line interface from within the game for remote administration and databse inqueries.
